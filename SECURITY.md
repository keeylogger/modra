# Security policy

## Supported versions

Modra is in **alpha**. Only the latest `1.x` line receives fixes. If you depend
on Modra in production (you brave soul), pin the exact patch version.

## Reporting a vulnerability

**Do not open a public issue for security reports.**

Instead, use GitHub's private vulnerability reporting flow:

1. Go to the [Security tab](../../security) of this repo.
2. Click **Report a vulnerability**.
3. Describe the impact, the reproduction steps, and (if you have one) the
   suggested mitigation.

You should hear back within 5 working days. Coordinated disclosure timelines
will be agreed case-by-case.
